# Telegram Channel Scanning + Full-Length Video Streaming

**Date:** 2026-03-13
**Status:** Approved

---

## Overview

Add two capabilities to the Desire Research Radar:

1. **Telegram channel scanning** — discover and configure Telegram channels via UI, scrape photos and videos using Pyrogram (MTProto), apply existing vision filter to photos, store video metadata for on-demand streaming.
2. **Full-length video streaming** — stream Telegram videos through a FastAPI proxy endpoint with Range header support so the HTML5 player can seek.

---

## Authentication

- Uses **Pyrogram** with a `StringSession` — no interactive auth at runtime.
- A one-time helper script (`scripts/telegram_auth.py`) handles phone → code → session string flow.
- Credentials stored in `.env`:
  ```
  TELEGRAM_API_ID=37478698
  TELEGRAM_API_HASH=de230ec6f07a43bf072590d128fc3c34
  TELEGRAM_SESSION=BQA...   # generated by auth helper
  TELEGRAM_SCAN_LIMIT=200   # messages per channel per scan pass
  ```
- The Pyrogram `Client` is a lazily-started module-level singleton in `app/sources/telegram.py`, reused across all requests.

## Database Schema — 2 New Tables

```sql
CREATE TABLE telegram_channels (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT NOT NULL UNIQUE,
    display_name TEXT NOT NULL,
    member_count INTEGER,
    description TEXT,
    enabled INTEGER NOT NULL DEFAULT 1,
    auto_discovered INTEGER NOT NULL DEFAULT 0,
    last_scanned_at TEXT,
    added_at TEXT NOT NULL
);

CREATE TABLE telegram_media (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    channel_username TEXT NOT NULL,
    message_id INTEGER NOT NULL,
    media_type TEXT NOT NULL,        -- 'photo' | 'video' | 'gif'
    width INTEGER,
    height INTEGER,
    duration INTEGER,                -- seconds; NULL for photos
    file_size INTEGER,
    caption TEXT,
    local_path TEXT,                 -- populated for photos; NULL for videos
    passes_filter INTEGER NOT NULL DEFAULT 1,
    posted_at TEXT,
    created_at TEXT NOT NULL,
    UNIQUE(channel_username, message_id)
);
```

## Backend

### `app/sources/telegram.py`

- Module-level `_client: pyrogram.Client | None = None` singleton.
- `get_client(settings) -> Client` — lazy init with `StringSession`.
- `discover_channels(query: str, settings) -> list[dict]` — calls OpenAI to suggest 10 channel usernames for the query, resolves each via Pyrogram `get_chat()`, returns list of `{username, display_name, member_count, description}` for valid channels only.
- `scan_channel(username, since_message_id, db, settings) -> dict` — iterates messages via `iter_messages()`, for each `photo` message: downloads to `data/screenshots/`, applies `passes_vision_filter()`, inserts to `telegram_media` and optionally `screenshots`. For each `video`/`gif`: stores metadata only. Returns `{photos, videos, skipped}`.

### `app/api/telegram.py`

Router at `/api/telegram`:

| Method | Path | Description |
|--------|------|-------------|
| GET | `/channels` | List all configured channels |
| POST | `/channels` | Add channel by `{username}` |
| DELETE | `/channels/{username}` | Remove channel |
| POST | `/channels/discover` | AI discovery — body `{query}` → candidates |
| POST | `/scan` | Trigger background scan of all enabled channels |
| GET | `/scan/status` | `{running, last_result}` |
| GET | `/media` | Paginated browse — params: `media_type`, `channel`, `limit`, `offset` |
| GET | `/media/{id}/stream` | Range-aware streaming proxy |

### Streaming endpoint

`GET /api/telegram/stream/{media_id}`:
- Looks up `telegram_media` row to get `channel_username` + `message_id`.
- Re-fetches the message via Pyrogram to get a fresh `file_id` (Telegram file_ids expire).
- Streams via `client.stream_media(message, offset, limit)` with `StreamingResponse`.
- Parses `Range` header to support seek; responds with `206 Partial Content`.

## Frontend

### MediaPage changes

- Add `"telegram"` to source-type filter. When active, renders the Telegram sub-UI instead of the screenshot grid.
- Telegram sub-UI has two tabs: **Channels** and **Videos**.

### Channels tab

- Text input + "Add channel" button (add by username).
- "Discover channels" button → opens a modal with a text prompt. User types topic (e.g. "gay sex videos"). Returns AI-suggested candidates as checkboxable cards (`display_name`, `member_count`, `description`). User checks desired ones → "Add selected" inserts them.
- List of configured channels: `display_name`, `@username`, member count, `enabled` toggle, remove button, last-scanned timestamp.
- "Scan now" button + scan status (running indicator, last-result summary).

### Videos tab

- Same masonry grid as screenshots.
- Each card: dark thumbnail placeholder with ▶ play icon + duration badge + channel name + caption snippet.
- Click → opens **VideoPlayerModal** (full-screen overlay):
  - HTML5 `<video controls>` with `src="/api/telegram/media/{id}/stream"`
  - Caption text below.
  - Close button.

### Photos from Telegram

Flow into the existing screenshot grid as `source = "telegram"` — no separate view, unified with DDG captures.

## Vision Filter Integration

- Applied to **photos** inline during `scan_channel()` — same `passes_vision_filter()` from `app/vision_filter.py`.
- For **videos**: vision filter is skipped at scan time (can't evaluate video frames). Videos are shown unfiltered in the Videos tab. The retroactive "Scan & clean library" button on MediaPage does not apply to videos.
- Photos that fail the filter are not stored (same fail-fast pattern as DDG captures).

---

## Error Handling

- Pyrogram client not configured (no session string) → API returns `503` with `{"error": "Telegram not configured"}`.
- Channel not found / private → `discover` returns it as `{valid: false}`, not added.
- Stream endpoint: if message no longer exists or file expired → `404`.
- All scan errors are logged and counted in `last_result.errors`; scan continues to next channel.
